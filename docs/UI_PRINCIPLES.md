# ShinGiTai UI Principles

Status: Batch 7.1 Identity System
Owner: ArticSakuraTech / ShinGiTai Holding Groupe
Last updated: 2026-07-03

## Purpose

This document defines how ShinGiTai interfaces should feel and behave.

It applies first to ShinGiTai Language, then later to ShinGiTai Hub, Forge and OpenAI.

## Principle 1 - Simple Outside, Powerful Inside

The application may have complex systems behind it, but the user should see a clear path.

Every screen should answer:

- Where am I?
- What can I do now?
- What is recommended next?
- What did the AI do for me?

## Principle 2 - Dashboard First

The dashboard is mission control.

It should not be a random collection of cards.

It should guide the user toward:

- continuing learning,
- practicing with AI,
- reviewing flashcards,
- finishing daily goals,
- seeing progress.

## Principle 3 - AI As Mentor, Not Toy

AI features should feel like a serious mentor.

Avoid making AI feel like a chatbot slapped onto a product.

Preferred AI patterns:

- guided next steps,
- clear explanations,
- error correction,
- memory-aware suggestions,
- study plans,
- conversation practice.

## Principle 4 - Premium, Not Loud

Premium does not mean adding more glow everywhere.

Premium means:

- good spacing,
- readable hierarchy,
- intentional motion,
- consistent surfaces,
- good empty states,
- calm confidence.

## Principle 5 - Ecosystem Ready

Every global UI pattern should work in the future Hub.

Examples:

- user profile menu,
- app switcher,
- notifications,
- settings,
- AI provider selection,
- account/billing entry points.

## Principle 6 - No Prototype Smell

Avoid visible signs of generated prototype code:

- inconsistent naming,
- mixed product names,
- placeholder copy,
- random icons,
- unfinished empty states,
- inconsistent spacing.

## Principle 7 - Clear Feedback

Every user action should have feedback.

Examples:

- loading state,
- success state,
- error state,
- disabled state,
- optimistic update if safe.

## Principle 8 - Strong Empty States

Empty states should guide users.

Bad:

> No data.

Good:

> No flashcards yet. Generate your first deck from today's lesson.

## Principle 9 - Mobile And Desktop Awareness

Language should eventually work as:

- web app,
- desktop app,
- mobile app,
- Hub module.

Layouts should avoid assumptions that only work on one screen size.

## Principle 10 - Trust Before Monetization

Premium and payment features should feel natural.

The app should first prove value, then ask for upgrade.

Good premium placement:

- unlock advanced AI tutor,
- voice practice,
- custom avatars,
- longer memory,
- offline desktop mode,
- advanced progress analytics.

Avoid aggressive paywalls during first experience.

## Application To ShinGiTai Language

Priority product screens:

1. Landing page
2. Auth
3. Onboarding
4. Dashboard
5. AI Tutor
6. Lessons
7. Flashcards
8. Grammar
9. Settings
10. Account/Premium

## Future Application To ShinGiTai Hub

The same principles should support:

- app launcher,
- update manager,
- AI center,
- plugin marketplace,
- account center,
- documentation hub.
